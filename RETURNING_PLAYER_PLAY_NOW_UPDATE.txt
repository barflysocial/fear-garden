Fear Garden update: returning-player Play Now behavior

Implemented:
- Play Now now works for players who are already checked in, even if the game has already started.
- Late check-in is still blocked for new players when Late Check-In After Start = No.
- Returning players are no longer treated like brand-new late arrivals.
- My RSVP lookup now prioritizes an already checked-in live session.
- If a player is already checked in, Play Now returns them to the current phase served by the game state:
  - Waiting Room
  - Case Briefing
  - Investigation
  - Final Accusation
  - Case Closed

Technical notes:
- Server lookup now prioritizes checked-in live sessions before upcoming sessions.
- /api/checkins allows already checked-in players through but blocks new late arrivals when late check-in is disabled.
- /api/access/join accepts rsvpLookup from the player page so free/shared-code returning players can be matched back to their RSVP.
- New late arrivals are blocked at join time when late check-in is disabled.
